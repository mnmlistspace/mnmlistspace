How to update Slackware Linux

Before starting the update process, it is a good habit to check Slackware ChangeLog. Check for critical packages important to your workâ€”research on Internet for potential problems and possible solutions. Be a proactive participant in the process, not just an observer.

Although the process of update takes some time, updating Slackware Linux is easy and hassle-free.

Let's start:

Run slackpkg update gpg to connect to a Slackware mirror and initialize your slackpkg's database. If this is your first update at all, you should get such an output. 


           # slackpkg update gpg
           You do not have any mirror selected in /etc/slackpkg/mirrors
	   Please edit that file and uncomment ONE mirror. Slackpkg only works with ONE mirror selected.

Edit the /etc/slackpkg/mirrors with an editor of your choice (I'm using vi for such kinds of tasks) and select a mirror close to your geolocation. Be careful with two super essential details:

  • There can be only one mirror server.
  • The file has two server lists. The first list is for Slackware64(or 32)-14.2 (a.k.a stable) installation,
    and the second one is for Slackware64-current.
    If you use a stable - DO NOT UNCOMMENT mirrors for current and vice versa.

Once you have selected your mirror and saved the /etc/slackpkg/mirrors, you can rerun the slackpkg update gpg.

The next step consists of updating modified packages. If you have any 3rd party packages installed on your system, please blacklist them so that slackpkg will not interfere with those applications.

Install the newly added packages to the Slackware distribution:


          # slackpkg install-new

Upgrade the whole system:


          # slackpkg upgrade-all
