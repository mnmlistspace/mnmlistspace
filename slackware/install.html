Slackware on a desktop - The Beginner's Guide

As a graphic designer in this digital era, I am forcing to use Microsoft Windows or macOS. A considerable part of my clients and almost all of my colleagues I work with expect me to use Adobe Creative Cloud. These requirements sometimes have their technical reasons, and sometimes they are just stupid twists. For this reason, I use my windows workstation only when I have to deal with Adobe products. For everything else, I grant in GNU/Linux and, more specifically, in Slackware Linux.

I have been using Slackware for everyday desktop tasks since 2002. Over the years, I have tried all the significant Linux distributions and some not-so-significant ones. I always go back to the good old Slackware. In my eyes and mind, he offers everything I need and works perfectly. For me, the combination of Thinkpad laptop and Slackware Linux is the greatest thing that can happen to a minimalist person in the IT industry.

Stable or Current?

Slackware Linux has two main branches. "Stable" realize and "Current." The stable branch is suitable for newcomers and those who need a stable workstation or reliable server. The current branch is the place where the dev team pushes the latest changes in the development process. I'm using the Slackware stable. If you need the current branch, I highly recommend checking alienbob's website.

Download slackware

First, let's get the installation files we need from Slackware's website. Slackware Mirror List uses MirrorBrain, which offers us two convenient options: mirror selection by network topology and geolocation by an estimated geographical distance of the mirrors to the client. Take a look at the list of: Found X mirrors which handle this country (XX). These are the files on the mirror servers suitable for you to download. Select one link and download the iso and the md5 files.


            $ cd /Downloads
            $ curl -O https://mirror.telepoint.bg/slackware/slackware-iso/slackware64-14.2-iso/slackware64-14.2-install-dvd.iso
            % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
            Dload  Upload   Total   Spent    Left  Speed
            9 2645M    9  247M    0     0   979k      0  0:46:04  0:04:18  0:41:46 1009k

Wait for iso to download and get the md5 file in the same directory:


            $ curl -O https://mirror.telepoint.bg/slackware/slackware-iso/slackware64-14.2-iso/slackware64-14.2-install-dvd.iso.md5
            % Total    % Received % Xferd  Average Speed   Time    Time     Time  Current
            Dload  Upload   Total   Spent    Left  Speed
            100    67  100    67    0     0    833      0 --:--:-- --:--:-- --:--:--  1015

Let's check the integrity of the downloaded iso file with md5sum:


            $ md5sum slackware64-14.2-install-dvd.iso > downloaded-iso.md5
            $

Now we need to compare the two files with sdiff:


            $ sdiff -s slackware64-14.2-install-dvd.iso.md5  downloaded-iso.md5
            $

In the Unix tradition, no news means good news. Once we have "no" output from the previous command, we can continue creating the USB. Plugin the USB stick and copy the disk image using the dd command. This command assumes the USB drive is recognizing as sdb (check dmesg to verify):


            # dd if=slackware64-14.2-install-dvd.iso.md5  of=/dev/sdb

Boot your laptop from the USB installer. This guide assumes you're booting via "BIOS compatibility mode" or "BIOS legacy mode" options. If you have UEFI mode, you should disable them.
